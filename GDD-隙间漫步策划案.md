# 隙间漫步（Gap Wander）—— 游戏策划案

---

## 一、游戏概述

### 1.1 基本信息

| 项目 | 内容 |
|------|------|
| **游戏名称** | 隙间漫步（Gap Wander） |
| **引擎** | Godot 4.7 |
| **类型** | 2D 俯视角 Roguelike 共斗 |
| **核心参考** | 《挺进地牢》(Enter the Gungeon) + 《黑夜君临》(Elden Ring Nightreign) |
| **视角** | 2D Top-Down |
| **目标平台** | PC (Steam) |
| **玩家人数** | 1-3人（单人 / 在线联机） |
| **开发周期** | 12周 |

### 1.2 核心体验

> 「三人成行，在随机生成的地牢迷宫中限时探索、战斗、收集，在黑夜降临前变强，合力击败夜之Boss，存活至最后。」

玩家在准备房间集结后，进入一个由多个房间随机连接构成的 2D 地牢。每个房间可能是安全的通路房（收集资源），也可能是需要战斗的据点房（击败敌人获得三选一强化）。游戏有两天的时间循环，每天下午开始缩圈，夜晚缩至一个安全房间并刷新 Boss。击败两天的 Boss 后进入最终 Boss 战。

### 1.3 核心循环

```
准备房间 → 探索房间 → 收集资源/战斗强化 → 缩圈逃生 → 击败夜之Boss 
    → 第二天 → 探索 → 缩圈 → 夜之Boss → 最终Boss → 结算奖励
```

## 二、玩法系统详细设计

### 2.1 地图系统

#### 2.1.1 地图结构

- **基础形状**：10×10 方形网格，或半径为 10 的伪圆形区域
- **单位**：每个格子 = 1 个房间
- **房间总数**：约 80-100 个房间（伪圆形）或 100 个房间（方形）
- **房间连接**：相邻房间之间由通路连接（上下左右四向）

#### 2.1.2 房间类型

**A. 通路房（Passage Room）**
- 所有通往相邻房间的通道**从一开始就敞开**
- 无强制战斗要求
- 可能包含：
  - 可破坏物件（木桶、箱子）→ 掉落金币/消耗品
  - 随机散落的消耗品（血瓶、护甲片）
  - 低概率出现的 NPC 商人
- 约占房间总数的 60%

**B. 据点房（Stronghold Room）**
- 进入后，通往其他房间的出口**初始封锁**
- 必须击败房间内刷新的特定敌人才会解锁出口
- **进入的入口保持敞开**，玩家可以撤退
- 击败特定敌人后掉落「**命运之球**」（三选一强化）
- 约占房间总数的 35%

**据点房子类：**

| 子类型 | 特点 | 地图标识 | 占比 |
|--------|------|----------|------|
| **标准据点** | 2-4个普通敌人 + 1个精英敌人 | 普通图标 | 25% |
| **强敌房** | 1个类Boss强敌，掉落更高级的命运之球 | 特殊图标（骷髅/皇冠） | 5% |
| **宝库房** | 无敌人，但解谜/钥匙开门后可获高级奖励 | 宝箱图标 | 3% |
| **事件房** | 随机事件（献祭、赌博、交易等） | 问号图标 | 2% |

**C. Boss 房（特殊）**
- 每天夜晚缩圈后，在安全房间内刷出夜之Boss
- 最终Boss有独立的大型战斗场景
- 不计入普通房间占比

### 2.2 时间与缩圈系统

#### 2.2.1 一天的时间线

```
白天（探索期）        下午（缩圈期）        夜晚（Boss战）
|——————————————————|——————————→ ←————————|——————————————|
 自由探索            圈从地图边缘          圈缩至1个房间
 清理房间            向中心收缩            刷新夜之Boss
 收集资源            圈外玩家受百分比伤害    击败Boss进入下一天
```

#### 2.2.2 缩圈规则

- **触发时机**：每天的游戏时间到达"下午"节点
- **收缩方式**：安全区从地图边缘向中心逐圈收缩，类似大逃杀
- **收缩终点**：收缩至**1个房间大小**（该房间为通路房，不选中据点房）
- **圈外伤害**：玩家在安全区外每秒受到最大生命值 5%-10% 的伤害
- **缩圈速度**：约 60-90 秒完成一次完整收缩
- **视觉表现**：暗色雾气从边缘涌入，安全区边界有明显光效

#### 2.2.3 Boss 刷新

- 缩圈完成后，安全房间内所有原有内容被清除
- 刷新夜之Boss（第1天为 Tier1 Boss，第2天为 Tier2 Boss）
- Boss 被击败后，进入下一天（或触发最终Boss战）

### 2.3 战斗系统

#### 2.3.1 玩家能力

- **移动**：8方向自由移动（WASD / 左摇杆）
- **瞄准**：鼠标瞄准 / 右摇杆瞄准
- **射击**：鼠标左键 / RT 使用主武器
- **闪避翻滚**：空格 / A键，短暂无敌帧
- **主动道具**：Q键 / LB 使用当前装备的主动道具
- **交互**：E键 / X键 拾取物品、开门、与NPC对话

#### 2.3.2 武器系统

| 类型 | 特点 | 示例 |
|------|------|------|
| **手枪类** | 均衡，弹夹适中 | 标准手枪、连发手枪 |
| **步枪类** | 远距离，高精度 | 突击步枪、狙击枪 |
| **霰弹类** | 近距离，扇形散射 | 双管霰弹、泵动霰弹 |
| **能量类** | 特殊效果，消耗能量 | 激光枪、电击枪 |
| **投掷类** | 弧形弹道，范围伤害 | 手榴弹、燃烧瓶 |
| **近战类** | 近距离高伤，可格挡弹幕 | 剑、长矛 |

#### 2.3.3 命运之球（三选一强化）

击败据点房特定敌人后掉落一个可交互的光球，交互后弹出三选一界面：

- **选项A/B/C**：各展示一件武器或一个被动升级
- **稀有度**：普通（白）、稀有（蓝）、史诗（紫）、传说（金）
- **强敌房**的命运之球稀有度下限提升（至少稀有）
- **选择后**：立即获得对应武器/升级，其余两个选项消失

### 2.4 进度系统

#### 2.4.1 局内进度（单局）

```
游戏开始 → 第1天白天 → 第1天下午(缩圈) → 第1天夜晚(Boss) 
    → 第2天白天 → 第2天下午(缩圈) → 第2天夜晚(Boss) → 最终Boss战 → 结算
```

- 默认 **2天 + 最终Boss**
- 可通过局外道具扩展为 3-4 天

#### 2.4.2 局外进度（Meta Progression）

- **遗物系统**：局内收集的特殊道具可带出，在准备房间的「遗物台」投入
- **遗物效果示例**：
  - 「破碎的日晷」：本局变为 3 天
  - 「暗影之契」：替换最终 Boss 为更强大的变体
  - 「幸运兔脚」：所有命运之球变为四选一
  - 「商人之友」：商店价格降低 30%
- **角色解锁**：达成特定条件解锁新角色
- **难度阶梯**：通关后解锁更高难度

### 2.5 联机系统

#### 2.5.1 准备房间

- 玩家进入准备房间（大厅）
- 可创建房间（Host）或加入房间（Join）
- 1-3人：单人也可直接开始
- 在准备房间中：
  - 选择角色
  - 投入遗物（每个玩家可投1个）
  - 查看队友装备
  - 发起准备就绪 → 开始游戏

#### 2.5.2 联机规则

- 主机权威（Host Authoritative）网络模型
- 敌人血量随玩家人数动态调整
- 玩家倒地后可被队友救援（长按交互键，3秒）
- 全队倒地 = 游戏结束
- 命运之球的选择：每个玩家独立选择自己的奖励

## 三、12周开发计划

### Week 1：最小玩法循环（MVP）

**目标：一周内跑通完整的最小玩法循环**

| 模块 | 内容 |
|------|------|
| **玩家控制器** | 8方向移动 + 鼠标瞄准 + 射击 + 翻滚闪避（无敌帧） |

```

[Boss战] → 击败Boss
     ↓
[胜利！] → 显示结算 → 返回准备房间（可重新开始）
```

## 五、技术架构建议

### 5.1 Godot 4.7 技术选型

| 需求 | 方案 |
|------|------|
| **2D 渲染** | Node2D + TileMapLayer（新版 TileMap） |
| **物理碰撞** | CharacterBody2D（玩家）、AnimatableBody2D/StaticBody2D（场景） |
| **弹幕系统** | Area2D + 自定义 BulletManager |
| **网络** | ENetMultiplayerPeer（Godot 内置）+ RPC |
| **UI** | Control 节点 + Theme |
| **存档** | ConfigFile 或自定义 JSON 读写 |
| **音效** | AudioStreamPlayer2D + AudioBus 分层 |

### 5.2 项目结构规范

- 使用 `class_name` 定义基类
- 信号（Signal）解耦系统间通信
- 资源文件（.tres / .res）管理配置数据
- 使用 `@export` 暴露可调参数，方便策划调试
